# ADR-0001：ctxops 项目方向

## 背景

这个话题起点很实际：

如何让团队在大型项目中更高效地使用 AI，而不要求每个人都先彻底熟悉整个代码仓库？

调研后可以确认几件事：

- AI 编码需求已经很强
- instruction files 和仓库本地上下文正在成为共识
- 市场上已有 coding agent、repo packager 和规则标准
- 但仍缺少团队级的上下文运维能力

## 决策

把 `ctxops` 定义成一个**文档优先、CLI first、仓库本地化**的 ContextOps 工具链。

核心关注点是：

- 上下文 schema
- 上下文组合
- 多工具渲染
- 新鲜度和安全校验

而不是去做一个通用 coding agent。

## 后果

正向影响：

- 与通用 agent 明确区分
- 更贴近真实团队痛点
- 更适合开源冷启动

负向影响：

- 热度可能不如通用 agent 产品
- 需要靠 demo 和文档证明价值

## 备选方案

### 备选 1：做通用 coding agent

被拒绝，因为执行层已经非常拥挤，竞争面太广。

### 备选 2：只发布规范

被拒绝，因为只有规范没有产品闭环。

### 备选 3：先做云端平台

被拒绝，因为仓库权限、同步和托管复杂度来得太早。
